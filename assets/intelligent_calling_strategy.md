# 智能调用策略与资产评分系统

## 一、资产评分系统

### 1. 初始评分规则

每条新资产的初始评分为 **3分**（满分5分）。

**评分维度：**

| 维度 | 权重 | 说明 |
|---|---|---|
| 相关性 | 30% | 与目标话题的匹配度 |
| 独特性 | 25% | 是否有新颖的表达或视角 |
| 可用性 | 20% | 是否易于在多个平台复用 |
| 效果 | 15% | 历史表现（点赞、转发、评论） |
| 安全性 | 10% | 是否存在风险或合规问题 |

### 2. 评分更新机制

**人工复盘评分（V1入口）：** 当内容发布后，根据实际表现进行人工评分调整。

**自动评分升级（V2规划）：** 根据平台数据（点赞率、转发率、评论率、粉丝增长）自动调整评分。

**评分调整规则：**

- 高表现内容（点赞率>10%）：评分+0.5，最高5分
- 中等表现内容（点赞率5-10%）：评分保持或+0.2
- 低表现内容（点赞率<5%）：评分-0.3，最低1分
- 风险内容（被举报/删除）：评分直接降至1分

### 3. 使用次数追踪

每次调用资产时，自动增加 `usage_count`。

**使用次数影响：**

- 使用次数越多，下次调用优先级越低（避免重复）
- 同一周内避免调用同一Hook结构超过1次
- 同一月内避免调用同一案例超过2次

## 二、智能调用策略

### 1. 优先级排序算法

**调用优先级 = 评分 × (1 - 使用次数系数) × 平台匹配度**

其中：
- **评分**：1-5分
- **使用次数系数**：`min(usage_count / 10, 0.8)`（使用10次后系数达到0.8上限）
- **平台匹配度**：该资产是否标记为适用该平台（1.0或0.5）

**示例计算：**

```
资产A：评分4.5，使用3次，适用视频号
优先级 = 4.5 × (1 - 3/10) × 1.0 = 4.5 × 0.7 = 3.15

资产B：评分3.0，使用0次，适用视频号
优先级 = 3.0 × (1 - 0/10) × 1.0 = 3.0 × 1.0 = 3.0

结论：优先调用资产A（尽管评分低，但新鲜度高）
```

### 2. 内容生成时的调用规则

**每条内容必须包含：**

- 1个Hook（来自Hook库，优先调用高分且低使用次数的）
- 1个Anchor（来自Anchor库，必须与身份一致，优先调用经验锚）
- 1个CTA（来自CTA库，根据内容类型选择轻/中/重引流）

**调用流程：**

```
1. 根据内容类型（框架/案例/观点/误区）确定所需Hook类别
2. 在该类别中，按优先级排序，取Top 3
3. 随机选择Top 3中的一个（增加多样性）
4. 更新该Hook的usage_count
5. 记录调用时间和内容ID
```

### 3. 去重与覆盖规则

**同一Hub的21条内容必须覆盖以下6类：**

| 内容类型 | 数量 | 说明 |
|---|---|---|
| 观点类 | 4条 | 核心观点、认知冲突、反常识 |
| 框架类 | 4条 | 方法论、决策框架、流程 |
| 案例类 | 4条 | 真实案例、反面案例、对比案例 |
| 误区类 | 3条 | 常见错误、认知误区、陷阱 |
| 清单类 | 3条 | 检查清单、行动清单、评估清单 |
| 答疑类 | 3条 | Q&A、常见问题、深度解答 |

**QA质检时检查：**

- [ ] 是否覆盖了所有6类？
- [ ] 每类是否至少2条？
- [ ] 是否有重复的Hook或Anchor？
- [ ] 同周内是否有相似度过高的内容？

## 三、爆款回流机制

### 1. 爆款标记流程

当某条内容被标记为"爆款"时（通常是点赞率>15%或转发率>5%），触发Learning流程：

**爆款定义：**

- 视频号：点赞率>10%，或转发>100
- 小红书：点赞率>15%，或收藏>50，或评论>20
- 公众号：阅读率>30%，或转发>20

### 2. 爆款拆解与资产化

**提取以下元素：**

- **标题结构**：分析标题的冲突点、长度、用词特点
- **开头句式**：提取黄金3秒的表达方式
- **论证路径**：核心观点→支撑论证→案例→结论
- **结尾CTA**：分析CTA的位置、表述、引导力

**写入结构模板库：**

```json
{
  "template_id": "TEMPLATE_20260301_001",
  "source_content_id": "爆款内容ID",
  "platform": "video_number",
  "structure": {
    "title_pattern": "为什么___都___，其实___",
    "opening_hook": "黄金3秒提问式",
    "body_structure": "观点→论证→案例→结论",
    "closing_hook": "收尾钩子",
    "cta_type": "轻引流"
  },
  "performance": {
    "likes_rate": 0.12,
    "shares": 150,
    "comments": 45
  },
  "template_score": 4.8
}
```

### 3. 资产评分提升

爆款内容中的所有资产（Hook、Anchor、CTA）评分自动提升：

- 爆款Hook：评分+1.0（最高5分）
- 爆款Anchor：评分+0.8
- 爆款CTA：评分+0.8
- 爆款结构模板：新增评分4.8

## 四、长期进化与复利机制

### 1. 资产库增长预期

| 时间 | 预期资产数 | 预期高分资产占比 |
|---|---|---|
| 第1个月（4周） | 80-100 | 20% |
| 第3个月（12周） | 240-300 | 35% |
| 第6个月（24周） | 480-600 | 50% |
| 第12个月（52周） | 1000+ | 65% |

### 2. 内容质量提升预期

**随着资产库扩大，新生成内容的质量应呈指数级提升：**

- **第1-4周**：大量新资产，质量参差不齐，平均评分3.0
- **第5-12周**：高分资产开始积累，平均评分3.5
- **第13-24周**：高分资产占主导，平均评分4.0
- **第25周+**：形成稳定的高质量内容生产体系，平均评分4.3+

### 3. 生产效率提升预期

**内容生成时间应随资产库完善而缩短：**

- **初期**：从Hub到21条内容需要8-10小时（包括LLM调用、人工审核）
- **中期**（第8周）：缩短至4-6小时
- **成熟期**（第24周+）：缩短至1-2小时（大量复用高分资产）

### 4. 复利效应体现

**可复利的三个层面：**

1. **资产层**：高分资产被反复调用，每次调用都为新内容增加价值
2. **模板层**：爆款结构被提炼成模板，指导新内容生成
3. **系统层**：整个系统的评分精准度越来越高，调用策略越来越智能
